# Canvas Sprite Repair Pipeline

## 背景

画布里的视频到 Sprite 流程不能把“AI 修图”当成一次性透明抠图。当前算法抠图能给出位置、尺寸、帧序列和粗 alpha，但会出现绿边、黑边、锯齿、细节丢失等问题。AI 的价值不是替代抠图，而是参考原始帧把角色 RGB 细节补回来。

因此流程应拆成多个明确节点，每个节点只解决一个问题。

## 节点边界

### 1. Transparent Sequence

输入：

- 视频节点
- 抽帧和抠图参数

输出：

- 透明帧序列
- 每帧的粗 alpha
- 每帧的原始帧引用

职责：

- 抽帧
- 基础抠图
- 输出可预览的透明序列

不负责：

- AI 补绘
- 修复缺失角色细节
- 判断最终 alpha 是否干净

### 2. RGB Repaint Candidate

输入：

- 原始视频帧
- 当前算法抠图结果

输出：

- RGB 补绘候选

职责：

- 根据原始帧补回角色细节
- 修复绿幕残留、黑白边、锯齿、破洞和脏边
- 尽量保持与当前抠图相同的位置、比例和轮廓意图

不负责：

- 输出最终透明 PNG
- 生成可直接写回透明序列的 alpha
- 猜测或清理棋盘格背景

当前实现中，RGB 补绘候选只能预览和对比，不能直接应用到透明帧序列。

### 3. Alpha Refine

输入：

- 当前粗 alpha
- RGB 补绘候选
- 原始帧

输出：

- 精修 alpha mask

职责：

- 修复边缘 alpha
- 去除绿边、黑边、白边
- 保留半透明细节
- 输出可验证的真实 alpha 通道

验收：

- 输出尺寸必须等于目标帧尺寸
- 外部区域 alpha 为 0
- 主体区域 alpha 合理连续
- 不能把棋盘格或纯色背景当成透明

### 4. Composite Final Sprite

输入：

- RGB 补绘候选
- 精修 alpha mask
- 目标帧尺寸和定位信息

输出：

- 最终透明 sprite 帧

职责：

- 把 RGB 和 alpha 合成为真正透明 PNG
- 写回透明帧序列的新版本
- 保留旧版本用于对比和回退

验收：

- PNG 有真实 alpha 通道
- 画布尺寸与原帧一致
- 主体位置不漂移
- 可与原版本并排对比

## 工程原则

- 不用前端启发式规则猜透明背景。
- 不把模型返回图直接当最终透明资产。
- 不在同一个节点里混合 RGB 修复和 alpha 决策。
- 每一步都生成可追溯中间产物，失败时能看到问题发生在哪一层。
- 新能力通过模板策略或处理节点扩展，不把业务流程硬编码进画布核心。

## 当前实现状态

- `frame-sequence-list` 是模板策略节点，负责展示帧列表和候选版本。
- `AI 修图`按钮现在只生成 `rgb_repaint` 候选。
- `rgb_repaint` 候选不可直接应用到透明序列。
- 后续应新增 Alpha Refine / Composite 能力，产出 `canApply: true` 的候选后再允许写回。
