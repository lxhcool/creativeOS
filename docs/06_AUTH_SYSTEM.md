# 06 Auth System

## 登录注册方式

Creative OS 使用邮箱验证码统一登录 / 注册。

用户不需要区分登录和注册。

如果邮箱已存在：

```text
验证码验证成功 → 登录
```

如果邮箱不存在：

```text
验证码验证成功 → 自动注册 → 创建个人工作区 → 创建默认项目 → 登录
```

## 邮件服务

验证码发送使用 Resend。

## 用户流程

### 1. 获取验证码

用户输入邮箱，点击获取验证码。

系统：

1. 校验邮箱格式
2. 检查发送频率
3. 生成 6 位数字验证码
4. 保存验证码 hash
5. 使用 Resend 发送验证码

### 2. 验证验证码

系统：

1. 校验验证码是否存在
2. 校验是否过期
3. 校验是否已使用
4. 校验错误次数
5. 验证通过后登录或注册

### 3. 登录成功

系统创建 HttpOnly Session Cookie。

推荐：

```text
Session 有效期：30 天
活跃用户自动续期
```

## 后续登录体验

用户下次访问时，浏览器自动携带 Session Cookie。

如果 Session 有效，直接进入工作台，无需再次输入验证码。

## 重新登录场景

- 用户主动退出
- Session 过期
- Cookie 被清除
- 更换设备

## 安全规则

- 验证码不明文入库，只保存 hash
- 验证码 10 分钟过期
- 验证码只可使用一次
- 单邮箱 60 秒内只能发送一次
- 单邮箱 10 分钟最多 5 次
- 单 IP 10 分钟最多 20 次
- 验证码错误 5 次后失效
- 接口不得暴露邮箱是否已注册
